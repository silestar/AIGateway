# AIGateway (AGW) 项目目录结构规范

> 版本：1.0  
> 最后更新：2026-05-04

## 详细说明

### 1. 入口与命令

|路径|说明|
|---|---|
|`cmd/agw/main.go`|项目唯一入口，负责初始化配置、依赖注入、启动 Gin 服务。|
|`Makefile`|提供 `make build`, `make run`, `make test`, `make lint` 等便捷命令。|

### 2. 核心业务逻辑 (`internal/`)

`internal/` 下各模块遵循**接口定义 + 实现**模式，通过 `uber/dig` 进行依赖注入。

|目录|职责|
|---|---|
|`config/`|加载 `config.yaml`、`.env` 以及环境变量，并合并为统一配置结构体。|
|`consumer/`|消费者（API Key 使用者）的 CRUD、认证、配额管理以及权限分组绑定。|
|`channel/`|渠道的 CRUD、模型自动发现与映射、以及渠道下的总体状态维护。|
|`account/`|账号池核心逻辑：优先级排队、用户亲和（粘性）、故障降级、按需探测恢复。|
|`group/`|分配与路由引擎：渠道分组、消费者分组、模型存在性检查、分层确定性路由。|
|`proxy/`|高性能 HTTP 代理引擎，负责透明转发、流式处理、连接池复用。|
|`log/`|系统运行日志（`zap`，按日归档）和请求日志异步批量写入。|
|`stats/`|实时计数器（Redis）与日聚合统计表维护，提供数据给 API 及前端。|
|`i18n/`|后端国际化支持，根据请求头返回对应语言提示。|
|`plugin/`|插件生命周期管理：注册、启动、健康检查、钩子调用。|
|`crypto/`|全局加密/解密工具（AES-256-GCM），用于保护账号 Key、插件 Token 等。|
|`storage/`|数据库抽象层，提供 `Storage` 接口，实现 SQLite、MySQL、PostgreSQL 驱动。|

### 3. 可导出包 (`pkg/`)

这些包可以被外部项目引用，尤其插件 SDK 可独立发布。

|目录|说明|
|---|---|
|`adapter/`|定义 `ChannelAdapter` 接口，并提供 OpenAI、Claude、Gemini 等实现。|
|`sdk/`|插件开发 SDK（Go 语言），封装钩子注册、认证、HTTP 服务启动等。|
|`middleware/`|Gin 通用中间件，如 Recovery、CORS、请求限流、日志记录等。|

### 4. 前端 (`web/`)

- 使用 **Vue 3 + TypeScript + Naive UI + ECharts + vue-i18n**。
    
- `src/` 下按功能模块组织组件，`locales/` 存放 `zh-CN.json` 和 `en-US.json`。
    

### 5. 开发设计文档 (`docs/`)

**面向人群**：开发者、架构师、贡献者。

|路径|说明|
|---|---|
|`architecture/01-总体设计方案.md`|全局架构、技术选型、模块划分、部署方案。|
|`modules/02-账号池核心逻辑详细设计.md`|账号选择、粘性、故障降级、按需探测的完整规则。|
|`modules/03-插件系统接口规范.md`|插件协议、钩子定义、Go SDK、安全设计。|
|`modules/04-请求日志与统计聚合详细设计.md`|异步写入、聚合表、查询 API、生命周期管理。|
|`modules/05-模型自动发现与映射详细设计.md`|FetchModels 接口、API 设计、与路由协作。|
|`ui/06-WebUI详细页面原型.md`|页面路由、组件结构与交互说明。|
|`CHANGELOG.md`|按版本记录所有新增、变更、修复的条目。|
|`VERSION`|纯文本文件，仅包含当前版本号（如 `0.1.0`）。|

### 6. 用户教程站点 (`guide/`)

**面向人群**：系统部署者、日常运维者、最终用户。  
构建工具：**VitePress**（可部署为 `https://docs.agw.dev`）。

|文件|内容|
|---|---|
|`index.md`|项目简介、一键启动命令、快速验证。|
|`deployment.md`|Docker Compose / K8s / 二进制部署，数据库切换，Redis 配置。|
|`configuration.md`|`config.yaml` + `.env` 所有配置项详细说明。|
|`channels.md`|渠道创建、模型发现、模型映射、账号管理操作演示。|
|`consumers.md`|消费者创建、API Key 管理、权限分组、配额设置。|
|`groups.md`|渠道分组与消费者分组配置，权重与绑定关系。|
|`plugins.md`|插件安装、启用、配置、官方 Go SDK 快速上手。|
|`statistics.md`|仪表盘解读，用户/渠道分析，请求日志使用指南。|
|`api-reference.md`|管理 API 完整参考（消费者、渠道、账号、分组、统计）。|
|`faq.md`|常见部署问题、路由行为解释、账号恢复机制说明。|

### 7. 提案 (`proposals/`)

**面向人群**：核心开发团队。

- `README.md` 说明提案的创建、讨论、实施与归档流程。
    
- 进行中的新功能或重构方案直接放在 `proposals/` 根目录，文件名格式：`YYYYMMDD-简短描述.md`。
    
- 实施完成的方案移动到 `archive/` 内长期保存，作为决策历史。
    

### 8. 部署与配置

|文件|用途|
|---|---|
|`docker/Dockerfile`|多阶段构建镜像，打包 Go 后端与前端静态文件。|
|`docker/docker-compose.yml`|包含 `agw` 和 `redis` 服务，开箱即用。|
|`config.yaml.example`|基础配置模板，展示所有可配项及默认值。|
|`.env.example`|环境变量模板，列明 `SECRET_KEY`、DB、Redis 等变量，不含真实机密。|
|`.gitignore`|忽略 `.env`、`/data`、`/logs`、`web/dist`、`*.db` 等。|

### 9. 项目根文件

|文件|说明|
|---|---|
|`go.mod`|Go模块定义，管理依赖。|
|`README.md`|项目概览、特性、快速启动、文档索引。|

## 文件命名与组织原则

1. **文档**：使用中文名，前缀数字表示阅读顺序（仅 `docs/` 内）。
    
2. **代码包**：使用小写英文字母，不包含下划线或数字。
    
3. **提案**：以日期开头，便于排序。
    
4. **配置模板**：以 `.example` 结尾，防止真实密钥意外提交。
