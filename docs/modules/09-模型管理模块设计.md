# 模型管理模块设计

## 1. 概述

模型管理模块聚合所有活跃渠道的模型映射，提供统一的模型目录管理，并实现 OpenAI 兼容的 `/v1/models` 端点。

### 1.1 核心功能

- **自动同步**：从 `channel_models` 表全量聚合模型目录，去重后写入 `model_catalog`
- **可见性控制**：管理员可控制哪些模型对外暴露（`visible` 字段）
- **OpenAI 兼容**：`/v1/models` 端点返回标准 OpenAI 格式

### 1.2 模型分类

| 分类 | 判断条件 | 说明 |
|------|---------|------|
| 已选模型 | `display_model_name == actual_model_name` | 未做自定义映射的原始模型 |
| 自定义映射模型 | `display_model_name != actual_model_name` | 管理员自定义了别名 |

---

## 2. 数据库设计

### 2.1 `model_catalog` 表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | UINT | PK, AUTO_INCREMENT | 主键 |
| `model_name` | VARCHAR(100) | UNIQUE, NOT NULL | 去重后的 display_model_name |
| `is_mapped` | BOOL | NOT NULL, DEFAULT false | 是否为自定义映射模型 |
| `visible` | BOOL | NOT NULL, DEFAULT true | 是否在 /v1/models 返回 |
| `ref_count` | INT | NOT NULL, DEFAULT 0 | 引用该模型的活跃渠道数 |
| `created_at` | TIMESTAMP | AUTO_CREATE | 创建时间 |
| `updated_at` | TIMESTAMP | AUTO_UPDATE | 更新时间 |

---

## 3. 同步逻辑

### 3.1 同步触发时机

| 操作 | 触发方式 |
|------|---------|
| 渠道保存模型（`SaveModels`） | `go s.triggerModelsChange()` 异步回调 |
| 删除整个渠道（`Delete`） | `go s.triggerModelsChange()` 异步回调 |
| 渠道启用/禁用（`UpdateStatus`） | `go s.triggerModelsChange()` 异步回调 |
| 服务启动 | 同步调用 `SyncFromChannelModels()` |

### 3.2 同步流程（全量刷新）

```
SyncFromChannelModels():
  1. 从所有 status != 'disabled' 的渠道的 channel_models（status='enabled'）
     按 display_model_name 分组，得到当前有效的模型集合
     同时计算每个模型：
     - has_mapping: 是否存在 display != actual 的记录
     - ref_count: 引用该模型的渠道数
  
  2. 获取 model_catalog 现有记录

  3. 对比处理：
     - 新模型 → INSERT (visible=true, is_mapped=has_mapping)
     - 已存在且 catalog.is_mapped=false，当前 has_mapping=true → 更新 is_mapped=true
     - 已存在且 visible=false，当前 ref_count > 0 → 恢复 visible=true
     - 不在有效集合中的模型 → ref_count=0, visible=false

  4. 不删除记录——保留管理员的可见性配置
```

### 3.3 关键设计决策

- **全量刷新而非增量**：每次同步重新扫描所有活跃渠道，确保数据一致性
- **不删除记录**：模型从所有渠道移除时标记 `visible=false`，保留记录以便模型重新出现时恢复
- **不覆盖 is_mapped**：管理员可能手动调整过映射标记，同步时不覆盖已有的 `is_mapped=true`
- **异步回调**：通过 `onModelsChange` 回调触发同步，不阻塞主流程

## 4. API 端点

### 4.1 代理端点

#### `GET /v1/models`

OpenAI 兼容的模型列表端点。

**响应格式**：

```json
{
  "object": "list",
  "data": [
    {
      "id": "gpt-4o",
      "object": "model",
      "owned_by": "aigateway"
    }
  ]
}
```

**过滤条件**：`visible = true AND ref_count > 0`

### 4.2 管理端点

#### `GET /api/models/catalog`

返回完整的模型目录（含可见和隐藏的）。

**响应格式**：

```json
{
  "data": [
    {
      "id": 1,
      "model_name": "gpt-4o",
      "is_mapped": false,
      "visible": true,
      "ref_count": 3,
      "created_at": "2026-05-10T12:00:00Z",
      "updated_at": "2026-05-10T12:00:00Z"
    }
  ]
}
```

#### `PUT /api/models/catalog/:id/visibility`

切换模型可见性。

**请求体**：

```json
{
  "visible": false
}
```

**响应格式**：

```json
{
  "data": {
    "id": 1,
    "visible": false
  }
}
```

## 5. 前端页面

### 5.1 路由与菜单

- 路由：`/models`
- 侧边栏菜单：🤖 模型（位于"日志"和"插件"之间）

### 5.2 页面布局

左右两列布局：

| 左列 | 右列 |
|------|------|
| **已选模型**（`is_mapped=false`） | **自定义映射模型**（`is_mapped=true`） |
| 蓝色标签 | 橙色标签 |

每列包含：
- 卡片标题 + 计数标签
- 模型列表，每项显示：模型名标签 + 引用渠道数 + 可见性开关
- 开关带 tooltip 提示：开启/隐藏的效果说明

### 5.3 空状态

当没有任何模型数据时，显示引导提示："请先在渠道中配置模型映射，模型将自动同步到此处"

```
                   model_catalog 表
                    │            │

```
