# 模型管理模块设计

## 1. 概述

模型管理模块基于 `channel_models` 表直接管理模型可见性，实现 OpenAI 兼容的 `/v1/models` 端点。

### 1.1 核心功能

- **去重展示**：左列展示 `actual_model_name` 去重（上游模型），右列展示 `display_model_name` 去重（映射模型）
- **可见性控制**：通过 `upstream_visible` 和 `display_visible` 字段分别控制两列的可见性
- **OpenAI 兼容**：`/v1/models` 端点返回标准 OpenAI 格式

### 1.2 两列分类

| 列 | 数据源 | SQL | 说明 |
|------|------|------|------|
| 左列（上游模型） | `actual_model_name` 去重 | `SELECT DISTINCT actual_model_name ...` | 渠道配置的真实上游模型名 |
| 右列（映射模型） | `display_model_name` 去重 | `SELECT DISTINCT display_model_name ...` | 管理员自定义的映射别名 |

---

## 2. 数据库设计

### 2.1 `channel_models` 表（扩展字段）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `upstream_visible` | BOOL | NOT NULL, DEFAULT true | 左列勾选控制，是否作为上游模型暴露给 /v1/models |
| `display_visible` | BOOL | NOT NULL, DEFAULT true | 右列勾选控制，是否作为映射别名暴露给 /v1/models |

> 同一行记录的两个 visible 字段独立控制：一行 `channel_models`（如 `display=gpt-preview, actual=gpt-4o`）可以 upstream_visible=true 且 display_visible=false，表示只暴露 gpt-4o 不暴露 gpt-preview。

---

## 3. 可见性控制逻辑

### 3.1 模型页面的操作

| 操作 | 实际执行 |
|------|---------|
| 左列勾选 `gpt-4o` | `UPDATE channel_models SET upstream_visible=true WHERE actual_model_name='gpt-4o'` |
| 左列取消 `gpt-4o` | `UPDATE channel_models SET upstream_visible=false WHERE actual_model_name='gpt-4o'` |
| 右列勾选 `gpt-preview` | `UPDATE channel_models SET display_visible=true WHERE display_model_name='gpt-preview'` |
| 右列取消 `gpt-preview` | `UPDATE channel_models SET display_visible=false WHERE display_model_name='gpt-preview'` |

### 3.2 /v1/models 返回逻辑

```
SELECT DISTINCT actual_model_name FROM channel_models
  JOIN channels ON ... WHERE ... AND upstream_visible = true
UNION
SELECT DISTINCT display_model_name FROM channel_models
  JOIN channels ON ... WHERE ... AND display_visible = true
```

最终去重合并返回。

## 4. API 端点

### 4.1 代理端点

#### `GET /v1/models`

OpenAI 兼容的模型列表端点。

**响应格式**：

```json
{
  "object": "list",
  "data": [
    {
      "id": "gpt-4o",
      "object": "model",
      "owned_by": "aigateway"
    }
  ]
}
```

**逻辑**：返回 `upstream_visible=true` 的 actual_model_name + `display_visible=true` 的 display_model_name（去重合并）。

### 4.2 管理端点

#### `GET /api/models/list`

返回双列模型数据。

**响应格式**：

```json
{
  "data": {
    "upstream": [
      { "actual_model_name": "gpt-4o", "visible": true, "ref_count": 3 }
    ],
    "display": [
      { "display_model_name": "gpt-preview", "visible": false, "ref_count": 2 }
    ]
  }
}
```

#### `PUT /api/models/upstream/visible`

设置上游模型可见性。

**请求体**：

```json
{ "model_name": "gpt-4o", "visible": false }
```

#### `PUT /api/models/display/visible`

设置映射模型可见性。

**请求体**：

```json
{ "model_name": "gpt-preview", "visible": true }
```

## 5. 前端页面

### 5.1 页面布局

左右两列布局：

| 左列 | 右列 |
|------|------|
| **上游模型**（actual_model_name 去重） | **映射模型**（display_model_name 去重） |
| 蓝色标签 | 橙色标签 |
| checkbox 勾选 + 全选 | checkbox 勾选 + 全选 |
| 引用渠道数 | 引用渠道数 |

保存时 diff 检测变更，只提交有变化的模型。
