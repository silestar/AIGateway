# 插件系统接口规范（完整版）

## 1. 设计原则

- **安全隔离**：插件以独立进程（Sidecar）运行，通过 HTTP 与主系统通信。主系统通过 Token 认证调用。
    
- **语言无关，Go 优先**：官方提供 Go SDK 及示例，社区可使用任何语言实现，但推荐 Go 以保持技术栈统一。
    
- **热插拔**：支持在线启用/禁用插件，无需重启主系统。
    
- **轻量高效**：插件进程启动快、资源消耗低，与主系统松耦合。

## 2. 通信协议

### 2.1 传输层

- 协议：HTTP/1.1（未来可扩展至 gRPC）
    
- 地址：`http://127.0.0.1:<plugin_port>`
    
- 内容类型：`application/json; charset=utf-8`

### 2.2 认证

每个插件注册后由主系统生成 `PLUGIN_AUTH_TOKEN`（加密存储于数据库），启动时以环境变量注入。主系统调用插件的所有请求均需携带 Header：

```text
Authorization: Bearer <plugin_auth_token>
```

插件必须验证 Token，不匹配返回 `401 Unauthorized`。

### 2.3 超时与重试

- 主系统调用插件钩子的超时默认 **5 秒**（可通过配置调整）。
    
- 超时或网络错误视为插件调用失败，主系统执行兜底策略（跳过该钩子或拒绝请求，由具体钩子决定）。
    
- 主系统不重试对插件的调用。

## 3. 插件描述文件 (manifest.json)

插件根目录必须包含此文件，供主系统识别和加载。

```json
{
  "name": "content-filter",
  "version": "1.0.0",
  "description": "过滤请求中的敏感词",
  "author": "dev@example.com",
  "binary": "plugin",
  "port": 9001,
  "hooks": ["pre_request"],
  "config_schema": {
    "type": "object",
    "properties": {
      "blocked_words": { "type": "string", "description": "逗号分隔的敏感词列表" }
    }
  }
}
```

- `binary`：插件可执行文件名（Go 编译产物常见默认值为 `plugin`）。
    
- `port`：插件监听的本地端口。
    
- `hooks`：插件订阅的钩子列表。
    
- `config_schema`：定义管理员可在 WebUI 配置的参数（JSON Schema），安装插件后由主系统传递。
    

## 4. 钩子（Hook）定义与调用格式

### 4.1 可用钩子

|钩子名称|触发时机|是否可修改|用途|
|---|---|---|---|
|`pre_request`|认证完成、路由之前|可修改请求|内容过滤、参数校验、自定义认证|
|`post_response`|上游响应返回、发送给客户端前|可修改响应|内容过滤、格式转换、加解密|
|`on_log`|请求结束后，日志写入前|不可修改|自定义计费、外部审计日志|
|`account_select`|账号选择前（`SelectAccount` 执行之前）|可过滤候选列表|预过滤候选账号列表（从列表中移除不可用账号），不直接指定账号|

### 4.2 通用调用格式

主系统向插件发送 POST 请求：

```text
POST /hook/<hook_name>
Content-Type: application/json
Authorization: Bearer <token>

{
  "consumer_id": 123,
  "consumer_name": "app1",
  "model": "gpt-4",
  "request": {
    "headers": { "content-type": "application/json" },
    "body": { ... }
  },
  "response": {                           // 仅在 post_response / on_log 存在
    "status_code": 200,
    "headers": { "content-type": "application/json" },
    "body": { ... }
  },
  "channel_id": 5,                       // 部分钩子携带
  "account_id": 12,
  "candidate_accounts": [                 // 仅 account_select 携带
    {"id":12, "priority":1, "status":"active"},
    {"id":15, "priority":2, "status":"active"}
  ],
  "config": { "blocked_words": "word1,word2" }
}
```

### 4.3 钩子响应规范

#### `pre_request`

- **放行**：`{ "action": "continue" }`，可选携带 `modified_request` 字段局部覆盖请求。
    
- **修改**：`{ "action": "continue", "modified_request": { "headers": {...}, "body": {...} } }`
    
- **拒绝**：`{ "action": "reject", "status_code": 403, "message": "内容被过滤" }`

#### `post_response`

- 返回 `{ "action": "continue", "modified_response": { "headers": {...}, "body": {...}, "status_code": 200 } }`
    
- 或 `{ "action": "reject", ... }`

#### `on_log`

- 异步调用，主系统不等待响应，返回内容被忽略。

#### `account_select`

- **不干预**：`{ "action": "use_default" }`，保持候选列表不变，主系统按默认优先级+粘性策略选择。
    
- **预过滤**：`{ "action": "filter", "exclude_ids": [12, 15] }`，从候选账号列表中移除指定账号。主系统在过滤后的列表上执行默认的优先级+粘性选择，保持路由的确定性。

> **设计说明**：`account_select` 钩子在账号池 `SelectAccount` 之前执行，作为预过滤步骤。插件不直接指定最终使用的账号，仅从候选列表中移除不想要的账号。这样既保留了插件的自定义能力，又不破坏路由引擎的确定性原则。

## 5. Go 官方 SDK

提供 `pkg/plugin/sdk` 包，简化插件开发。

### 5.1 SDK 方法

```go
// 获取必需环境变量，若不存在则 panic
MustGetEnv(key string) string

// 注册钩子处理函数
HandleHook(hookName string, handler HookHandler)

// 启动 HTTP 服务，自动挂载 /health、/hook/*
StartPlugin(port string, token string)

// 钩子处理函数签名
type HookHandler func(req HookRequest) HookResponse
```

### 5.2 示例插件：敏感词过滤

```go
package main

import (
    "strings"
    "github.com/your-ai-gateway/plugin-sdk"
)

func main() {
    token := sdk.MustGetEnv("PLUGIN_AUTH_TOKEN")
    port := sdk.MustGetEnv("PLUGIN_PORT")

    var blockedWords []string
    if config := sdk.MustGetEnv("PLUGIN_CONFIG"); config != "" {
        // 简单解析配置，实际可使用 JSON unmarshal
        blockedWords = strings.Split(config, ",")
    }

    sdk.HandleHook("pre_request", func(req sdk.HookRequest) sdk.HookResponse {
        userText := req.Request.Body["messages"].(string) // 根据实际模型结构调整
        for _, word := range blockedWords {
            if strings.TrimSpace(word) != "" && strings.Contains(userText, word) {
                return sdk.Reject(403, "包含敏感词")
            }
        }
        return sdk.Continue()
    })

    sdk.StartPlugin(port, token)
}
```

## 6. 插件生命周期

### 6.1 注册与安装

- 管理员上传 ZIP 压缩包（包含 `manifest.json` 和二进制文件）。
    
- 系统解压至 `plugins/<name>/`，读取 `manifest.json` 并存入数据库。
    
- 插件状态为 `installed`。

### 6.2 启动

- 管理员点击“启用”，主系统执行 `binary` 文件，作为子进程运行。
    
- 注入环境变量：
    
    - `PLUGIN_AUTH_TOKEN`
        
    - `PLUGIN_PORT`
        
    - `PLUGIN_CONFIG` (JSON 字符串，由前端表单生成)
        
- 主系统开始周期性健康检查。

### 6.3 健康检查

主系统每 10 秒 `GET /health`，期望返回 `{"status":"ok"}`。连续 3 次失败则标记 `unhealthy` 并自动禁用，输出告警日志。

### 6.4 停止与卸载

- 禁用：发送 `POST /admin/shutdown`（带鉴权），之后终止进程。
    
- 卸载：先禁用，再删除插件目录及数据库记录。

## 7. 安全与错误处理

- **网络隔离**：插件端口仅绑定 `127.0.0.1`。
    
- **资源限制**：推荐通过 Docker 容器运行插件，限制 CPU/内存。
    
- **钩子错误降级**：任何钩子调用失败（超时、5xx、401），主系统记录警告并跳过该插件。`pre_request` 或 `post_response` 拒绝错误会直接终止请求。
    
- **敏感信息保护**：插件不得明文记录 Token 或请求内容，日志由主系统统一控制脱敏。
    
- **加密存储**：插件 Token 在数据库中使用全局 `SECRET_KEY` 加密存储。

## 8. WebUI 管理功能

- **插件列表**：名称、版本、状态（运行/已停用/异常）、开关按钮。
    
- **上传安装**：拖拽 ZIP 上传。
    
- **配置编辑**：根据 `config_schema` 动态生成表单，保存后重启插件生效。
    
- **日志**：可选提供插件 stderr 采集查看（未来扩展）。

## 9. 插件分发与构建

- **开发**：插件作者使用 Go（或其他语言）编写，提供 `manifest.json` 和可执行文件。
    
- **打包**：`zip plugin.zip plugin manifest.json`
    
- **分发**：可通过文件上传，或未来对接插件市场。

本规范基于核心系统完全 Go 化设计，确保了高性能、轻量级和统一的技术栈，同时保留了对其他语言的开放性。
